Watchlist Configuration

Configure multiple tickers for autonomous trading in one file.

Overview

The watchlist.json file allows you to configure multiple tickers for autonomous trading without creating individual .idiot files for each. Edit this file and restart the backend to trade all configured tickers.

File Location

IdiotProof.Core\Data\watchlist.json

File Format

{ "description": "My Trading Watchlist", "enabled": true, "session": "RTH", "tickers": [ { "symbol": "NVDA", "quantity": 5, "name": "NVIDIA", "enabled": true }, { "symbol": "AAPL", "quantity": 10, "name": "Apple", "enabled": true }, { "symbol": "TSLA", "quantity": 3, "name": "Tesla", "enabled": false, "session": "Premarket" }, { "symbol": "SPY", "quantity": 20, "name": "S&P 500 ETF", "enabled": true } ] }

Field Descriptions

Root-Level Fields

Field Type Description
description string Optional description of the watchlist
enabled boolean Global on/off switch. If false, no tickers trade.
session string Default session: "RTH", "Premarket", "AfterHours", "Extended"
tickers array Array of ticker configurations

Ticker Fields

Field Type Required Description
symbol string Yes Ticker symbol (e.g., "NVDA")
quantity number Yes Number of shares to trade
name string No Friendly name for display
enabled boolean No Enable/disable this ticker (default: true)
session string No Override default session for this ticker

Configuration Examples

Simple Watchlist

{ "enabled": true, "session": "RTH", "tickers": [ { "symbol": "AAPL", "quantity": 10 }, { "symbol": "MSFT", "quantity": 8 }, { "symbol": "GOOGL", "quantity": 5 } ] }

Mixed Sessions

{ "enabled": true, "session": "RTH", "tickers": [ { "symbol": "AAPL", "quantity": 10 }, { "symbol": "TSLA", "quantity": 3, "session": "Premarket" }, { "symbol": "SPY", "quantity": 20, "session": "Extended" } ] }

Temporarily Disable Some Tickers

{ "enabled": true, "session": "RTH", "tickers": [ { "symbol": "AAPL", "quantity": 10, "enabled": true }, { "symbol": "NVDA", "quantity": 5, "enabled": false }, // Skip today { "symbol": "META", "quantity": 8, "enabled": true } ] }

Programmatic Access

// Load watchlist var watchlist = WatchlistManager.Load(); // Generate IdiotScript for each enabled ticker foreach (var script in WatchlistManager.GenerateScripts()) { // script = "Ticker(NVDA).Name("NVDA Auto").Session(IS.RTH).Qty(5).AutonomousTrading()" } // Print summary WatchlistManager.PrintSummary(); // Add or update a ticker WatchlistManager.AddOrUpdate("META", 15); // Disable without removing WatchlistManager.Disable("TSLA"); // Remove completely WatchlistManager.Remove("GME");

Best Practices

  • Start small: Begin with 2-3 tickers to validate the system before scaling up.
  • Use consistent quantities: Size positions based on your account value and risk tolerance.
  • Disable instead of remove: Keep problematic tickers in the list but disabled for future reference.
  • Session matching: Use the default session for most tickers, override only when needed.
  • Backup regularly: Keep copies of your watchlist.json before making changes.
