AI Advisor (ChatGPT)

Your AI trading analyst. Interprets natural language stock tips, evaluates entries against custom rules, and provides a "third opinion" alongside technical indicators.

What the AI Advisor Does

The AI Advisor uses OpenAI's GPT-4 to provide intelligent trading analysis. It's not a replacement for indicators - it's an additional layer that:

  • Interprets natural language stock tips from strategy-rules.json
  • Evaluates whether your custom rules are satisfied
  • Provides a confidence score and reasoning
  • Identifies risk factors you might miss
  • Suggests TP/SL adjustments based on market context

Configuration

Setting the OpenAI API Key

Option 1: Environment variable (recommended)

# PowerShell $env:OPENAI_API_KEY = "sk-..." # Or add to system environment variables for persistence

Option 2: settings.json

{ "ai": { "openaiApiKey": "sk-...", "model": "gpt-4", "enableAdvisor": true } }

Response Format

The AI returns structured responses that are parsed into an AIAnalysis object:

AIAnalysis Properties

Property Type Description
Action string "LONG", "SHORT", or "WAIT"
Confidence int 0-100 confidence level
Reasoning string One-sentence explanation
RuleStatus string "MET", "NOT_MET", or "NO_RULES"
RiskFactors List<string> Identified risks
TpSlAdvice string Suggested take profit / stop loss
IsUsable bool True if API call succeeded
AreRulesMet bool Shorthand for RuleStatus == "MET"

Built-in Trading Principles

The AI Advisor is pre-programmed with trading principles via its system prompt:

AI TRADING PRINCIPLES (System Prompt) ═══════════════════════════════════════════════════════════════════════════════ ENTRY SIGNALS ───────────────────────────────────────────────────────────────────────── • Score > +70 with confirming LSH → Strong LONG signal • Score < -70 with confirming LSH → Strong SHORT signal • Conflicting signals → WAIT for clarity CAUTION FLAGS ───────────────────────────────────────────────────────────────────────── • RSI > 80 or RSI < 20 → Caution for reversals • ADX < 20 → Weak trend, avoid trend trades • Low volume → Be skeptical of moves CUSTOM RULES HANDLING ───────────────────────────────────────────────────────────────────────── When user-defined strategy rules are provided: • If rules say "wait for breakout then pullback": → Only recommend entry AFTER both conditions occur • If rules specify support levels: → Verify price is above those levels • Rules are ADDITIONAL filters: → They work WITH indicators, not against them • A good setup needs BOTH: → Good indicators AND rule compliance • If rules say "no chasing": → Recommend WAIT if no pullback has occurred RESPONSE REQUIREMENTS ───────────────────────────────────────────────────────────────────────── • In reasoning, explicitly state whether custom rules are satisfied • Never hallucinate or make up data • If uncertain, recommend WAIT • Be concise and actionable

Use Cases

1. Entry Analysis

Primary use case - evaluate a potential trade entry:

var analysis = await advisor.AnalyzeEntryAsync( symbol: "AAPL", snapshot: currentIndicators, lshForecast: patternMatchResult, score: marketScore, learnedWeights: tickerWeights ); if (analysis.Action == "LONG" && analysis.Confidence >= 70 && analysis.AreRulesMet) { // Strong buy signal with rules satisfied }

2. Learning Method Analysis

Evaluate which learning method performs best:

var analysis = await advisor.AnalyzeLearningResultsAsync( symbol: "NVDA", methods: learningResults, totalTrainingBars: 500, totalValidationBars: 100 ); // AI recommends: "GENETIC" or "ENSEMBLE"

3. Trade Outcome Analysis

Learn from completed trades:

var feedback = await advisor.AnalyzeTradeOutcomeAsync( symbol: "TSLA", isLong: true, entryPrice: 250.00, exitPrice: 245.00, entrySnapshot: indicators, exitReason: "Stop loss hit" ); // Returns: "RSI was at 78 at entry, suggesting overbought conditions..."

Rate Limiting & Caching

To control API costs and avoid rate limits, the AI Advisor implements:

  • Minimum call interval: 2 seconds between API calls
  • Response caching: 5-minute cache for identical requests
  • Call counting: Track API usage with advisor.CallCount

Estimated Costs (GPT-4):

  • ~500-1000 tokens per analysis call
  • ~$0.02-0.03 per call
  • 100 calls/day ≈ $2-3/day

Accuracy Tracking

The AI Advisor tracks its own accuracy for self-improvement:

// Record actual outcome after trade advisor.RecordOutcome(analysis, wasCorrect: true); // Get accuracy statistics var (total, correct, accuracy) = advisor.GetAccuracyStats(); Console.WriteLine($"AI Accuracy: {accuracy:F1}% ({correct}/{total})");
