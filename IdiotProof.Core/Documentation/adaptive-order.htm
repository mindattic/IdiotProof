AdaptiveOrder

Smart dynamic order management that adjusts Take Profit and Stop Loss levels in real-time based on market conditions.

What is AdaptiveOrder?

AdaptiveOrder monitors market conditions continuously and dynamically adjusts your Take Profit and Stop Loss levels. Instead of static exit prices, your orders respond intelligently to changing momentum, trend strength, and market sentiment.

💡 Key Insight

AdaptiveOrder requires you to set initial TakeProfit and/or StopLoss values. It modifies these dynamically but needs starting points for calculation.

Adaptive Behavior by Score

Score Range Market State Take Profit Stop Loss
+70 to +100 Strong Bullish Extend +25% to +75% Tighten (protect gains)
+30 to +70 Moderate Bullish Keep original Keep original
-30 to +30 Neutral/Choppy Reduce 10-25% Widen (allow bounce)
-70 to -30 Moderate Bearish Reduce 25-50% Keep original
Below -70 Strong Bearish EMERGENCY EXIT

Adaptive Modes

AdaptiveOrder offers three configuration modes that control how aggressively the system adjusts your TP/SL levels.

MODE COMPARISON ═══════════════════════════════════════════════════════════════════════════════ Setting │ CONSERVATIVE │ BALANCED │ AGGRESSIVE ═════════════════════════╪════════════════╪════════════╪═══════════════ MaxTakeProfitExtension │ 25% │ 50% │ 75% MaxTakeProfitReduction │ 60% │ 50% │ 30% MaxStopLossTighten │ 30% │ 50% │ 60% MaxStopLossWiden │ 40% │ 25% │ 15% EmergencyExitThreshold │ -60 │ -70 │ -80 MinScoreChangeForAdjust │ 10 │ 15 │ 20 ─────────────────────────┴────────────────┴────────────┴─────────────── CONSERVATIVE: Protects gains quickly, allows wider stops to avoid noise, exits sooner on bearish signals. Best for volatile stocks. BALANCED: Standard risk/reward, moderate adjustments in both directions. Good default for most situations. AGGRESSIVE: Lets winners run longer, tighter stops to protect capital, stays in longer during drawdowns. Best for trending stocks.

Usage Examples

// Default (Balanced mode) Ticker(AAPL) .Entry(150) .TakeProfit(160) .StopLoss(145) .AdaptiveOrder() // Conservative mode - protect gains Ticker(TSLA) .Entry(250) .TakeProfit(270) .StopLoss(240) .AdaptiveOrder(IS.CONSERVATIVE) // Aggressive mode - let winners run Ticker(NVDA) .Entry(500) .TakeProfit(550) .StopLoss(480) .AdaptiveOrder(IS.AGGRESSIVE)

Multiplier Calculation

TAKE PROFIT MULTIPLIER FORMULA ═══════════════════════════════════════════════════════════════════════════════ Score Range │ Multiplier Formula ═══════════════════╪══════════════════════════════════════════════════════ 70 to 100 │ 1.0 + (MaxExtension × (score-70)/30) │ → Score 100: TP extends up to +75% (aggressive) ───────────────────┼────────────────────────────────────────────────────── 30 to 70 │ 1.0 (no change) │ → TP stays at original value ───────────────────┼────────────────────────────────────────────────────── -30 to 30 │ 1.0 - (0.15 × (30-score)/60) │ → Slight reduction (up to 15%) ───────────────────┼────────────────────────────────────────────────────── -70 to -30 │ 0.85 - ((MaxReduction/2-0.15) × (-30-s)/40) │ → Moderate reduction ───────────────────┼────────────────────────────────────────────────────── < -70 │ 1.0 - MaxReduction │ → Maximum reduction (emergency) STOP LOSS MULTIPLIER FORMULA ═══════════════════════════════════════════════════════════════════════════════ Score Range │ Multiplier Formula ═══════════════════╪══════════════════════════════════════════════════════ 70 to 100 │ 1.0 + (MaxTighten × (score-70)/30) │ → Multiplier > 1.0 = TIGHTER stop (closer to entry) ───────────────────┼────────────────────────────────────────────────────── 0 to 70 │ 1.0 (no change) │ → SL stays at original distance ───────────────────┼────────────────────────────────────────────────────── -50 to 0 │ 1.0 - (MaxWiden × (-score/50) × 0.5) │ → Multiplier < 1.0 = WIDER stop (further from entry) ───────────────────┼────────────────────────────────────────────────────── < -50 │ 1.0 - (MaxWiden × 0.5) │ → Maximum widening to avoid noise stops

Concrete Example: CIGL Trade

CIGL ADAPTIVE TRADE EXAMPLE ═══════════════════════════════════════════════════════════════════════════════ Entry: $4.15 | Original TP: $4.80 | Original SL: $3.90 Profit Range: $0.65 | Loss Range: $0.25 Mode: AGGRESSIVE MARKET SCORE BREAKDOWN ───────────────────────────────────────────────────────────────────────── VWAP Score: +80 (price 4% above VWAP) × 15% = +12.0 EMA Score: +100 (price above all EMAs) × 20% = +20.0 RSI Score: -30 (RSI at 75, overbought) × 15% = -4.5 MACD Score: +90 (bullish + strong histogram) × 20% = +18.0 ADX Score: +100 (ADX 63 with +DI > -DI) × 20% = +20.0 Volume Score: +60 (1.6x average volume) × 10% = +6.0 ───────────────────────────────────────────────────────────────────────── TOTAL MARKET SCORE: +71.5 → Rounded to +72 TP CALCULATION (Score +72, Aggressive Mode) ───────────────────────────────────────────────────────────────────────── Score in range 70-100, so we EXTEND Take Profit MaxExtension (Aggressive) = 75% = 0.75 TP Multiplier = 1.0 + (0.75 × (72-70)/30) = 1.0 + (0.75 × 2/30) = 1.0 + 0.05 = 1.05 Original Profit Range = $4.80 - $4.15 = $0.65 New Profit Range = $0.65 × 1.05 = $0.6825 Adjusted TP = $4.15 + $0.6825 = $4.83 (up from $4.80) SL CALCULATION (Score +72, Aggressive Mode) ───────────────────────────────────────────────────────────────────────── Score in range 70-100, so we TIGHTEN Stop Loss MaxTighten (Aggressive) = 60% = 0.60 SL Multiplier = 1.0 + (0.60 × (72-70)/30) = 1.0 + (0.60 × 2/30) = 1.0 + 0.04 = 1.04 Original Loss Range = $4.15 - $3.90 = $0.25 New Loss Range = $0.25 / 1.04 = $0.24 (tighter = smaller range) Adjusted SL = $4.15 - $0.24 = $3.91 (up from $3.90, tighter protection) RESULT ───────────────────────────────────────────────────────────────────────── Original: Entry $4.15 → TP $4.80 (+15.7%) | SL $3.90 (-6.0%) Adjusted: Entry $4.15 → TP $4.83 (+16.4%) | SL $3.91 (-5.8%) → TP extended slightly, SL tightened for better risk/reward

Adaptive TP Feedback Loop

One of the most powerful features: the system extends TP when momentum is strong, then contracts it when momentum fades, allowing the price to eventually meet the target.

TP FEEDBACK LOOP VISUALIZATION ═══════════════════════════════════════════════════════════════════════════════ Price ↑ $165 ─ ─ ─ ─ ─ ─ ─ ─ Extended TP (score 90+) │ ╱ $163 ─│─ ─ ─ ─ ─ ╱─ ─ TP following momentum │ ╱ $161 ─│─ ─ ─ ╱─ ─ ─ ─ TP reducing as RSI overbought │ ╱ ╲ $160 ─│─ ╱─ ─ ─ ─ █ ← FILLED HERE (TP came down to meet price) │╱ ╱ $158 ─│─ ─ ─ ─ ╱─ Price path │ ╱ $155 ─│─ ─ ╱ │ ╱ $150 ─█╱─ ─ ─ ─ ─ Entry │ └──────────────────────────────────────────────────────────→ Time T=0 T=5 T=10 T=15 T=20 TIMELINE EXAMPLE ═══════════════════════════════════════════════════════════════════════════════ Entry: $150 | Original TP: $160 | Profit Range: $10 Mode: AGGRESSIVE (75% max extension) T=0 Price $151, Score +85 (strong momentum) → TP Multiplier = 1.375 → TP extended to $163.75 T=5 Price $158, Score +90 (very strong, approaching original TP) → TP Multiplier = 1.50 → TP extended to $165.00 → Price would have hit $160 but TP is now $165! System extended TP during strong momentum, then contracted it when momentum faded, allowing fill at optimal time.

RSI Overbought Detection

RSI is crucial for detecting when momentum is exhausted:

RSI SCORE CONTRIBUTION ═══════════════════════════════════════════════════════════════════════════════ RSI Value │ Score Contribution │ Effect on TP ═══════════╪══════════════════════╪══════════════════════════════════════ 30 or less │ +100 (oversold) │ Extend (bullish bounce expected) 40 │ +25 │ Slight extend 50 │ 0 (neutral) │ No change 60 │ +25 │ Slight extend 70 │ 0 (threshold) │ No change 75 │ -17 │ Start reducing TP 80 │ -33 │ Reduce TP 85 │ -50 │ Significant reduction 90+ │ -67 to -100 │ Maximum reduction How it works: ───────────────────────────────────────────────────────────────────────── As price rapidly approaches TP: → RSI rises toward 70+ (overbought) → RSI score contribution becomes NEGATIVE → Total market score drops → TP multiplier reduces → TP target comes down to meet the price → Trade exits at optimal time!

Emergency Exit

When the market score drops below the emergency threshold, AdaptiveOrder triggers an immediate exit to protect capital.

⚠️ Emergency Exit Thresholds
  • Conservative: Score ≤ -60
  • Balanced: Score ≤ -70
  • Aggressive: Score ≤ -80
