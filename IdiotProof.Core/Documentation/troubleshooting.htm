Troubleshooting

Solutions to common problems and error messages.

Connection Issues

Error: "Cannot connect to TWS"

Symptoms:

  • Application fails to start trading
  • Console shows "Connection refused" or timeout errors

Solutions:

  1. Verify TWS is running:
    Open Trader Workstation and log in. It must be running before IdiotProof starts.
  2. Check API settings in TWS:
    File → Global Configuration → API → Settings
    • ☑ Enable ActiveX and Socket Clients
    • ☑ Socket port: 7497 (paper) or 7496 (live)
    • ☑ Allow connections from localhost only
  3. Verify port in settings.json:
    { "ibkr": { "host": "127.0.0.1", "port": 7497, // Match TWS setting "clientId": 1 } }
  4. Check firewall:
    Ensure Windows Firewall allows IdiotProof and TWS to communicate.

Error: "Client ID already in use"

Cause: Another application or instance is connected with the same Client ID.

Solution: Change clientId in settings.json to a unique number (e.g., 2, 3, etc.).

Order Issues

Order not executing

Common causes:

Symptom Likely Cause Solution
Order pending but not filling Limit price too far from market Use market orders or adjust limit
"Outside RTH" rejection Order placed outside market hours without permission Add .OutsideRTH() to strategy
Insufficient funds Account has insufficient buying power Reduce quantity or add funds
Order rejected - PDT Pattern Day Trader restriction Account needs $25K+ or fewer day trades

Take Profit not triggering

Check these:

  1. TP level set correctly:
    For longs, TP must be ABOVE entry. For shorts, BELOW entry.
  2. AdaptiveOrder extending TP:
    If using AdaptiveOrder, TP may be dynamically extended. Check logs for adjusted levels.
  3. Price never reached:
    The price may have approached but not touched the TP level.

Indicator Issues

Indicators showing NaN or zero

Cause: Insufficient warm-up time. Indicators need historical data.

INDICATOR WARM-UP TIMES ═══════════════════════════════════════════════════════════════════════════════ Indicator Bars Needed Start Backend Early By ───────────────────────────────────────────────────────────────────────── EMA(9) 9 10 minutes EMA(21) 21 25 minutes EMA(200) 200 3+ hours ADX(14) 28 30 minutes RSI(14) 15 20 minutes MACD(12,26,9) 35 40 minutes Bollinger(20) 20 25 minutes ───────────────────────────────────────────────────────────────────────── Solution: Start backend 30+ minutes before trading

Conditions never triggering

Debugging steps:

  1. Check logs:
    View Logs/session_*.txt for indicator values and condition states.
  2. Verify condition values:
    IsAdxAbove(25) - Is ADX actually above 25? Check the dashboard.
  3. Sequential evaluation:
    Conditions evaluate in order. If an earlier condition isn't met, later ones don't matter.

Strategy Issues

Strategy not loading

Check these:

  1. File location:
    Scripts must be in IdiotProof.Core/Scripts/ with .idiot extension.
  2. Syntax errors:
    Check for typos. Commands need parentheses: IsAboveVwap() not IsAboveVwap.
  3. Missing Ticker():
    Every strategy must start with Ticker("SYMBOL").

Parse error messages

Error Meaning Fix
"Unknown command: XYZ" Command not recognized Check spelling, use canonical names
"Expected parameter" Command requires parameter Add required value: Entry(150)
"Invalid number" Parameter should be numeric Use number: TakeProfit(160)
"Unclosed string" Missing closing quote Complete string: Name("My Strategy")

AI Component Issues

AI Advisor not responding

Check these:

  1. API key configured:
    Set OPENAI_API_KEY environment variable or in settings.json:
    { "ai": { "openaiApiKey": "sk-..." } }
  2. Internet connection:
    AI Advisor requires internet access to reach OpenAI API.
  3. API quota:
    Verify your OpenAI account has available credits.

LSTM predictions seem wrong

This is expected sometimes:

  • LSTM provides probabilistic guidance, not certainty
  • Predictions improve with more training data
  • Max impact is ±25 points on Market Score - it's one input among many
  • After major news events, historical patterns may not apply

To reset LSTM:

Delete Profiles/{TICKER}.lstm.json to force retraining.

Performance Issues

High CPU usage

Causes and solutions:

  • Too many tickers: Each ticker runs calculations. Limit watchlist to essential symbols.
  • LSTM training: Training runs every 15 minutes and may spike CPU. This is temporary and normal.
  • Debug logging: Reduce log verbosity in settings.json.

Memory growing over time

Cause: Candle and indicator history accumulates.

Solution: The system automatically trims history beyond 500 candles. If memory is still high, restart the backend daily.

Checking Logs

Logs are the key to debugging. Find them in Logs/session_*.txt.

Getting Help

When reporting issues, include:

  1. The complete .idiot strategy file
  2. Relevant log sections from Logs/session_*.txt
  3. Your settings.json (with API keys removed)
  4. Steps to reproduce the issue
  5. Expected vs actual behavior
