Order Management

Complete guide to order types, position management, and trade execution in IdiotProof.

Order Direction

IdiotScript supports both long and short positions with explicit direction commands.

Command Action Description
Order() BUY Opens a LONG position (default)
Order(IS.LONG) BUY Explicitly opens a LONG position
Order(IS.SHORT) SELL Opens a SHORT position
Long() BUY Alias for Order(IS.LONG)
Short() SELL Alias for Order(IS.SHORT)

Examples

// Long position (default)
Ticker(AAPL)
    .Entry(150)
    .Order()              // Opens long position
    .TakeProfit(160)

// Short position
Ticker(SPY)
    .Entry(500)
    .Order(IS.SHORT)     // Opens short position
    .TakeProfit(490)
    .StopLoss(505)

Order Configuration

Following the single-responsibility pattern, each order aspect is configured with its own chained method.

ORDER CONFIGURATION CHAIN ═══════════════════════════════════════════════════════════════════════════════ Ticker(AAPL) │ ├── .Entry(150) ← Entry condition (price level) ├── .Order(IS.LONG) ← Direction (LONG or SHORT) ├── .Quantity(100) ← Number of shares ├── .PriceType(Price.VWAP) ← Execution price type ├── .OrderType(Limit) ← Market or Limit order ├── .OutsideRTH() ← Allow entry outside RTH ├── .TakeProfit(160) ← Take profit level ├── .StopLoss(145) ← Stop loss level ├── .TrailingStopLoss(2) ← Trailing stop ($ or %) └── .AdaptiveOrder() ← Dynamic TP/SL adjustment

Configuration Commands

Command Purpose Default
Quantity(n) Number of shares to trade 1
PriceType(type) Price for execution Price.Current
OutsideRTH() Allow entry outside RTH true
TakeProfitOutsideRTH() Allow TP outside RTH true

Stock Price Tiers & Auto-Quantity

When you omit .Quantity() or set Quantity(0), IdiotProof automatically calculates position size based on the stock's price tier. This ensures appropriate position sizing across different price levels.

STOCK PRICE TIER SYSTEM ═══════════════════════════════════════════════════════════════════════════════ ┌─────────────┬───────────────┬─────────────────┬─────────────────────────┐ │ TIER │ PRICE RANGE │ TARGET POSITION │ EXAMPLE QTY │ ├─────────────┼───────────────┼─────────────────┼─────────────────────────┤ │ PREMIUM │ $500+ │ ~$1,000 │ 2 shares @ $500 │ ├─────────────┼───────────────┼─────────────────┼─────────────────────────┤ │ BLUE CHIP │ $100 - $500 │ ~$1,000 │ 5 shares @ $200 │ ├─────────────┼───────────────┼─────────────────┼─────────────────────────┤ │ MID-CAP │ $25 - $100 │ ~$600 │ 12 shares @ $50 │ ├─────────────┼───────────────┼─────────────────┼─────────────────────────┤ │ SMALL-CAP │ $5 - $25 │ ~$350 │ 35 shares @ $10 │ ├─────────────┼───────────────┼─────────────────┼─────────────────────────┤ │ PENNY │ $1 - $5 │ ~$200 │ 80 shares @ $2.50 │ ├─────────────┼───────────────┼─────────────────┼─────────────────────────┤ │ MICRO │ <$1 │ ~$100 │ 200 shares @ $0.50 │ └─────────────┴───────────────┴─────────────────┴─────────────────────────┘ WHY TIERS MATTER: ───────────────────────────────────────────────────────────────────────── Premium/Blue Chip ($100+) • Larger position ($1,000) - more stable, established companies • Lower share count - each share represents significant value • Typically lower volatility % but larger $ moves Mid-Cap/Small-Cap ($5-$100) • Medium position ($350-$600) - balanced risk/reward • Moderate share count for good scaling • Mix of established and growth companies Penny/Micro (<$5) • Smaller position ($100-$200) - higher risk, protect capital • Higher share count but limited $ exposure • More volatile, higher potential loss per trade

Usage

// Auto-quantity based on price tier
Ticker(NVDA)                    // ~$900 = Premium tier
    .AutonomousTrading()        // Auto-calculates: ~1 share (~$900 position)

Ticker(AAPL)                    // ~$185 = Blue Chip tier
    .AutonomousTrading()        // Auto-calculates: ~5 shares (~$925 position)

Ticker(SOFI)                    // ~$12 = Small-Cap tier
    .AutonomousTrading()        // Auto-calculates: ~29 shares (~$350 position)

// Override with explicit quantity
Ticker(NVDA)
    .Quantity(10)              // Explicit: 10 shares regardless of tier
    .AutonomousTrading()
ℹ️ Tier Philosophy

The tier system follows a "prestige" model: higher-priced stocks get larger positions because they typically represent more established, less volatile companies. Lower-priced stocks get smaller positions to limit exposure to higher-risk plays.

Exit Management

IdiotProof provides multiple exit mechanisms that can be used individually or in combination.

EXIT TYPES ═══════════════════════════════════════════════════════════════════════════════ TAKE PROFIT (TakeProfit) Price │ ████████████ TP Hit │ ____/ │ ____/ TP ───┼─────────────────────────────────────────────────── │ ____/ Entry │ __█__/ └─────────────────────────────────────────────────────────── STOP LOSS (StopLoss) Entry │ __█__\ │ \____ SL ───┼─────────────────────────────────────────────────── │ \____ │ \____████████████ SL Hit └─────────────────────────────────────────────────────────── TRAILING STOP LOSS (TrailingStopLoss) Price │ Peak │ ____/\ │ ____/ \ ← Price drops │ ____/ \ Entry │ _/ \ │ \ Trail │ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─\─█─ Trail Stop Hit │ Trail follows \ │ price up but \____ │ never down └─────────────────────────────────────────────────────────── EXIT STRATEGY (ExitStrategy) │ Exit at IS.BELL │ ▼ │ __________/\__________/\____█____ │ │ Entry │ Close position │ at session end └─────────────────────────────────────────────────────────── 9:30 3:00 3:59 4:00

Exit Commands

TakeProfit(price)

Close position when price reaches target profit level.

Ticker(AAPL)
    .Entry(150)
    .TakeProfit(160)      // Close at $160 for profit

StopLoss(price)

Close position when price falls to stop loss level.

Ticker(AAPL)
    .Entry(150)
    .StopLoss(145)        // Close at $145 to limit loss

TrailingStopLoss(distance)

Trailing stop that follows price up but never down. Distance can be in dollars or percentage. 

Ticker(AAPL)
    .Entry(150)
    .TrailingStopLoss(5)  // Trail $5 below highest price

ExitStrategy(time)

Close position at specified time or condition.

Ticker(AAPL)
    .Entry(150)
    .ExitStrategy(IS.BELL)           // Exit at session bell

// Exit only if profitable
Ticker(AAPL)
    .Entry(150)
    .ExitStrategy(IS.BELL).IsProfitable()  // Exit at bell if in profit

Session Bell Times (IS.BELL)

IS.BELL resolves to 1 minute before the current session ends, allowing clean exits before market close.

Session IS.BELL Resolves To Market Close
Premarket 9:29 AM 9:30 AM (RTH open)
RTH (Regular) 3:59 PM 4:00 PM
After Hours 7:59 PM 8:00 PM

Explicit session bells are also available:

  • IS.PREMARKET.BELL → 9:29 AM
  • IS.RTH.BELL → 3:59 PM
  • IS.AFTERHOURS.BELL → 7:59 PM

Position Closing

To close positions manually or create exit-only orders:

Command Action Use Case
CloseLong() Submits SELL order Close existing long position
CloseShort() Submits BUY order Cover existing short position

Repeat Strategy

Use Repeat() to restart a strategy after it completes (via TP, SL, or TSL). This allows the strategy to re-enter when conditions are met again. 

Ticker(AAPL)
    .Entry(150)
    .TakeProfit(155)
    .StopLoss(147)
    .Repeat()              // After exit, watch for next entry

Complete Order Examples

1. Full Long Strategy

Ticker(AAPL)
    .Name("AAPL Breakout Long")
    .Session(IS.RTH)
    .Quantity(50)
    .Entry(150)
    .Order(IS.LONG)
    .IsAboveVwap()
    .IsEmaAbove(9)
    .IsMomentumAbove(0)
    .TakeProfit(160)
    .StopLoss(145)
    .TrailingStopLoss(3)
    .AdaptiveOrder(IS.BALANCED)
    .Repeat()

2. Short Strategy with Time Exit

Ticker(SPY)
    .Name("SPY Short to Close")
    .Session(IS.RTH)
    .Quantity(100)
    .Entry(505)
    .Order(IS.SHORT)
    .IsBelowVwap()
    .IsEmaBelow(21)
    .IsMacdBearish()
    .TakeProfit(495)
    .StopLoss(510)
    .ExitStrategy(IS.BELL).IsProfitable()

3. Simple Autonomous

Ticker(NVDA)
    .Name("NVDA Auto")
    .Session(IS.RTH)
    .Quantity(10)
    .AutonomousTrading()
