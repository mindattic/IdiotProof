IdiotScript Language Reference

The complete guide to IdiotScript - a domain-specific language for defining trading strategies in a fluent, readable syntax.

What is IdiotScript?

IdiotScript is a Domain-Specific Language (DSL) designed specifically for defining trading strategies. It uses a fluent interface pattern where methods can be chained together to create readable, declarative strategy definitions.

💡 Design Philosophy

IdiotScript follows the Single-Responsibility Pattern: each method does exactly one thing. Methods have at most ONE parameter, making strategies easy to read and modify. 

// A complete IdiotScript strategy
Ticker(AAPL)
    .Name("AAPL Momentum Play")
    .Session(IS.RTH)
    .Quantity(25)
    .Entry(185)
    .IsAboveVwap()
    .IsEmaAbove(9)
    .IsMomentumAbove(0)
    .Order(IS.LONG)
    .TakeProfit(192)
    .StopLoss(182)
    .AdaptiveOrder()

Execution Flow

IdiotScript conditions are evaluated sequentially like a state machine, not as a single simultaneous AND condition. Each condition must be met before the next is evaluated.

Command Categories

1. Configuration Commands

Setting up the strategy. Order doesn't matter within this group.

Command Description Example
Ticker(symbol) Stock symbol to trade. Required. Ticker(NVDA)
Name(name) Friendly name for the strategy. Name("Gap and Go")
Session(session) Trading session. Options: IS.RTH, IS.PREMARKET, IS.AFTERHOURS, IS.EXTENDED Session(IS.RTH)
Quantity(n) Number of shares to trade. Quantity(10)

2. Entry Conditions

Conditions that must be met before order execution. Order matters!

Command Description
Entry(price) Trigger when price reaches this level
Breakout() Price breaks above recent high
Pullback() Price pulls back after breakout
IsAboveVwap() Price is above VWAP
IsBelowVwap() Price is below VWAP
IsGapUp(pct) Stock gapped up X% from previous close
IsGapDown(pct) Stock gapped down X%
IsEmaAbove(period) Price above EMA
IsEmaBelow(period) Price below EMA
IsAdxAbove(val) ADX above threshold (trend strength)
IsDiPositive() +DI > -DI (bullish directional)
IsRsiOversold(val) RSI ≤ threshold (e.g., 30)
IsRsiOverbought(val) RSI ≥ threshold (e.g., 70)
IsMacdBullish() MACD > Signal line
IsMomentumAbove(val) Momentum ≥ threshold
IsVolumeAbove(mult) Volume ≥ multiplier × average

3. Order Commands

The action to take when conditions are met.

Command Description
Order(IS.LONG) Open a long position (buy)
Order(IS.SHORT) Open a short position (sell short)
Order() Defaults to IS.LONG
Long() Alias for Order(IS.LONG)
Short() Alias for Order(IS.SHORT)

4. Exit Conditions

How to exit the position.

Command Description
TakeProfit(price) Exit at target price
StopLoss(price) Exit if price drops to level
TrailingStopLoss(pct) Trailing stop percentage
ExitStrategy(time) Time-based exit
IsProfitable() Only exit if position is profitable

5. Smart Order Management

AI-powered order adjustments.

Command Description
AdaptiveOrder() Dynamic TP/SL adjustment (balanced)
AdaptiveOrder(IS.AGGRESSIVE) Maximize profits, tighter stops
AdaptiveOrder(IS.CONSERVATIVE) Protect gains, wider stops
AutonomousTrading() Full AI control of entry/exit
Repeat() Restart strategy after exit

Canonical vs Alias Names

⚠️ Use Canonical Names

The parser accepts both canonical (full) and alias (short) forms, but the serializer always outputs canonical forms. Use canonical names for consistency.

Canonical (Use This) Alias (Deprecated)
Quantity() Qty()
TakeProfit() TP()
StopLoss() SL()
TrailingStopLoss() TSL()
IsAboveVwap() AboveVwap()
ExitStrategy() ClosePosition()

Syntax Rules

  • Always include parentheses, even for commands without parameters: IsAboveVwap() not IsAboveVwap
  • Chain with dots: .Method().Method()
  • Strings use quotes: Name("My Strategy")
  • Numbers are bare: Entry(150) not Entry("150")
  • Constants are prefixed: IS.RTH, IS.LONG
