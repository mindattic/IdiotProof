Strategy Rules

Define custom trading rules in plain English that ChatGPT evaluates alongside technical indicators. Turn stock tips into actionable filters.

Overview

The strategy-rules.json file allows you to define trading rules in natural language. When you receive a stock tip like "wait for AAPL to break $150, then buy the pullback above $145", you can encode this as a strategy rule. ChatGPT will evaluate your entries against these rules.

File Location

IdiotProof.Core\Data\strategy-rules.json

Complete File Format

{ "description": "Custom Strategy Rules", "enabled": true, "rules": [ { "symbol": "CCHH", "enabled": true, "validFrom": "2026-02-07", "validUntil": "2026-02-10", "name": "Day 2 Breakout-Pullback", "rule": "Wait for breakout above $0.78, then pullback. Only enter if pullback holds above $0.70. RULE: No chasing - pullback entries only.", "levels": { "breakout": 0.78, "support": 0.70, "previousHigh": 0.78 }, "targets": [0.85, 1.00], "notes": "Day 2 watch - be patient for proper setup" } ] }

Field Reference

Field Type Required Description
symbol string Yes Ticker symbol (e.g., "CCHH")
enabled boolean No Enable/disable this rule (default: true)
validFrom string No Start date in EST (ISO 8601). Rule ignored before this date.
validUntil string No End date in EST (ISO 8601). Rule ignored after this date.
name string No Friendly name for the strategy
rule string Yes Plain text rule - ChatGPT interprets this
levels object No Key price levels (see below)
targets array No Target prices as array of numbers
notes string No Additional context for the AI

Available Level Fields

Level Description
breakoutPrice that must break for consideration (resistance)
supportPrice that pullback must hold above
previousHighPrevious high of day or resistance
previousLowPrevious low of day or support
vwapVWAP reference level
trigger1First trigger level before main breakout
microSupportMicro support for tighter stops
rangeLowRange low for choppy markets
rangeHighRange high for choppy markets

Date Validity

Rules can have validity windows using validFrom and validUntil. This is essential for stock tips that are date-specific.

DATE VALIDITY EXAMPLES ═══════════════════════════════════════════════════════════════════════════════ SCENARIO 1: Single-Day Tip ───────────────────────────────────────────────────────────────────────── You receive a tip on Friday for Monday's catalyst: "validFrom": "2026-02-09", ← Monday only "validUntil": "2026-02-09" SCENARIO 2: Multi-Day Watch ───────────────────────────────────────────────────────────────────────── Earnings runner that might squeeze for a few days: "validFrom": "2026-02-07", ← Friday through Tuesday "validUntil": "2026-02-11" SCENARIO 3: Prevent Backtesting Leakage ───────────────────────────────────────────────────────────────────────── Tip received today shouldn't affect historical backtests: "validFrom": "2026-02-08" ← Only applies from today forward ← No validUntil = no expiry RULE STATUS OVER TIME ───────────────────────────────────────────────────────────────────────── Feb 6th Feb 7th Feb 8th Feb 9th Feb 10th Feb 11th │ │ │ │ │ │ │ │◄──────────────────────►│ │ │ │ [NOT YET] [ACTIVE] [ACTIVE] [ACTIVE] [EXPIRED] │ │ │ ├───────────┴───────────────────────────────────────────────┤ validFrom: Feb 7 validUntil: Feb 10

Writing Effective Rules

Good Rule Examples

// Clear, actionable, specific "rule": "CONFIRMATIONS (in order): 1. Price breaks above $0.78 (previous HOD / double top) 2. DO NOT buy the breakout - wait for a pullback 3. Pullback must hold higher lows above $0.70 4. Only take entries if pullback forms clean bottoms above $0.70 RULE: No chasing. Pullback entries only."

Key Principles

  • Be explicit: "Wait for breakout THEN pullback" not "buy when it looks good"
  • Include levels: "$0.78 breakout", "$0.70 support" are specific
  • State what NOT to do: "No chasing", "No FOMO entries"
  • Order matters: Numbered steps help the AI understand sequence
  • End with clear rule: "RULE: No break = no trade"

Bad Rule Examples

// Too vague - ChatGPT can't evaluate "rule": "Buy when it looks like it's going up" // No specific levels "rule": "Enter on pullback" // Contradictory "rule": "Buy breakout but don't chase"

AI Response Format

When ChatGPT evaluates with custom rules, it returns a RULE_STATUS:

Status Meaning
MET All custom rules are satisfied. Entry aligns with your strategy.
NOT_MET Custom rules not satisfied. Wait for better setup.
NO_RULES No custom rules defined for this symbol.
