Learned Indicator Weights

IdiotProof learns optimal indicator weights for each ticker using genetic algorithms and gradient descent, maximizing predictive accuracy.

What Are Learned Weights?

Instead of using fixed indicator weights, IdiotProof learns the optimal weights for each ticker through historical analysis. Different stocks respond differently to indicators:

  • NVDA might respond strongly to momentum (MACD, RSI)
  • SPY might be more VWAP-sensitive
  • TSLA might weight ADX/DI heavily due to trending behavior

Learning Methods

IdiotProof uses two primary methods to optimize weights:

OPTIMIZATION METHODS ═══════════════════════════════════════════════════════════════════════════════ 1. GENETIC ALGORITHM ┌─────────────┐ │ Generation 0│ Random weight combinations └──────┬──────┘ │ ▼ ┌─────────────┐ │ Evaluate │ Test each combination on historical data │ Fitness │ Fitness = prediction accuracy × profit factor └──────┬──────┘ │ ▼ ┌─────────────┐ │ Select │ Keep top performers │ Best │ └──────┬──────┘ │ ▼ ┌─────────────┐ │ Crossover │ Combine best weights │ + Mutate │ Add small random variations └──────┬──────┘ │ ▼ ┌─────────────┐ │ Generation N│ Repeat until convergence └─────────────┘ 2. GRADIENT DESCENT Score = Σ (weight_i × indicator_i) For each trade: 1. Calculate prediction error 2. Compute gradient ∂Error/∂weight_i 3. Update: weight_i -= learning_rate × gradient_i Loss │ │\ │ \ │ \____ │ \____ │ \____ ← Converging to minimum │ ──────────── └───────────────────────────────▶ Iterations

Weight Files

Learned weights are stored in JSON files in the Data/ directory:

File Pattern Contents
{TICKER}.weights.json Indicator weights for this ticker
{TICKER}.history.json Trade history used for learning
{TICKER}.patterns.json Saved patterns for matching

Learning Process

LEARNING WORKFLOW ═══════════════════════════════════════════════════════════════════════════════ ┌───────────────────────────────────────────────────────────────────────┐ │ 1. COLLECT DATA │ │ │ │ For each trade (entry + exit): │ │ • Record all indicator values at entry │ │ • Record market score at entry │ │ • Record outcome (profit/loss, % gain) │ │ • Save to {TICKER}.history.json │ └───────────────────────────────────────────────────────────────────────┘ │ ▼ ┌───────────────────────────────────────────────────────────────────────┐ │ 2. ANALYZE CORRELATIONS │ │ │ │ Calculate correlation between each indicator and trade outcome: │ │ │ │ Indicator Winning Avg Losing Avg Correlation │ │ ──────────────────────────────────────────────────────── │ │ MACD Signal +0.82 -0.45 +0.89 (strong) │ │ RSI 62 48 +0.75 (good) │ │ VWAP Position +1.2% -0.8% +0.65 (moderate) │ │ ... │ └───────────────────────────────────────────────────────────────────────┘ │ ▼ ┌───────────────────────────────────────────────────────────────────────┐ │ 3. OPTIMIZE WEIGHTS │ │ │ │ Using correlation data, run optimization: │ │ • Genetic algorithm (50 generations, 100 population) │ │ • Or gradient descent (1000 iterations, LR=0.01) │ │ │ │ Objective: Maximize (Accuracy × ProfitFactor) │ └───────────────────────────────────────────────────────────────────────┘ │ ▼ ┌───────────────────────────────────────────────────────────────────────┐ │ 4. SAVE & APPLY │ │ │ │ • Save optimized weights to {TICKER}.weights.json │ │ • Market score calculator loads ticker-specific weights │ │ • Future trades use optimized weights │ └───────────────────────────────────────────────────────────────────────┘

Data Requirements

ℹ️ Minimum Data for Learning

Meaningful weight optimization requires sufficient historical data:

  • Minimum: 20 trades to start learning
  • Recommended: 50+ trades for stable weights
  • Optimal: 100+ trades for high confidence
Trade Count Weight Confidence Blending
< 20 Low 100% default weights
20-50 Moderate 50% learned + 50% default
50-100 Good 75% learned + 25% default
> 100 High 100% learned weights

Retraining Weights

Weights are automatically updated as new trades complete. You can also trigger manual retraining:

Automatic Retraining

  • Triggered every 10 completed trades
  • Uses rolling window of last 200 trades
  • Gradual blending to avoid sudden weight shifts

Manual Reset

To reset weights to defaults and start fresh, delete the weight file: 

Delete: Data/{TICKER}.weights.json
Delete: Data/{TICKER}.history.json

Best Practices

💡 Pro Tips
  • Let weights stabilize - Don't trust weights with <50 trades
  • Review periodically - Market conditions change, weights should too
  • Check correlations - If MACD weight is 0%, it's not predicting well for this ticker
  • Compare to defaults - Large deviations indicate ticker-specific behavior
⚠️ Overfitting Warning

Weights learned from too few trades may overfit to specific market conditions. The blending mechanism helps prevent this, but always maintain healthy skepticism with low trade counts.
