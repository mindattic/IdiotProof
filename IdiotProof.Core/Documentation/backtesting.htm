Backtesting

Test your strategies against historical data to validate performance before risking real capital.

What Is Backtesting?

Backtesting runs your trading strategy against historical market data to see how it would have performed in the past. While past performance doesn't guarantee future results, it helps validate strategy logic and identify potential weaknesses.

Data Requirements

  • Minimum: 20 trading days for basic testing
  • Recommended: 3-6 months for meaningful results
  • Optimal: 1+ year covering different market conditions

Running a Backtest

Basic Backtest

// Load strategy from .idiot file
var strategy = IdiotScriptParser.Parse(@"
    Ticker(NVDA)
    .IsAboveVwap()
    .IsEmaAbove(9)
    .Entry(150)
    .Order(IS.LONG)
    .TakeProfit(160)
    .StopLoss(145)
");

// Create backtester
var backtester = new Backtester(strategy);

// Run against historical data
var results = await backtester.RunAsync(
    startDate: DateTime.Parse("2025-01-01"),
    endDate: DateTime.Parse("2025-12-31")
);

// Print results
results.PrintSummary();

Drawdown Analysis

Drawdown measures the peak-to-trough decline in your account. Understanding drawdown helps you prepare psychologically and set appropriate position sizes.

DRAWDOWN VISUALIZATION ═══════════════════════════════════════════════════════════════════════════════ Equity Curve: $11,000 ┤ **** │ **** $10,500 ┤ **** │ **** **** $10,000 ┤ **** **** │ ****──────── Peak $9,500 ┤ **** │ **** $9,200 ┤ **** ── Trough (-8%) │ **** $9,500 ┤ **** │ **** └──────────────────────────────────────────────▶ Time |←── MAX DRAWDOWN: -8% ──→| |←── 12 DAYS TO RECOVER ──→|
⚠️ Drawdown Warning

In live trading, actual drawdowns are often 1.5-2x larger than backtested drawdowns due to slippage, gaps, and market impact. If your backtest shows 8% max drawdown, prepare for 12-15% in reality.

Comparing Strategies

Test multiple strategies against the same data to find the best performer: 

var comparison = new StrategyComparison();

comparison.AddStrategy("VWAP Breakout", vwapStrategy);
comparison.AddStrategy("EMA Crossover", emaStrategy);
comparison.AddStrategy("Momentum Only", momentumStrategy);

var results = await comparison.RunAsync(
    startDate: DateTime.Parse("2025-01-01"),
    endDate: DateTime.Parse("2025-12-31")
);

results.PrintComparisonTable();

Walk-Forward Testing

Walk-forward analysis helps detect overfitting by testing on out-of-sample data:

WALK-FORWARD ANALYSIS ═══════════════════════════════════════════════════════════════════════════════ Standard Backtest (In-Sample Only = High Overfitting Risk): ┌────────────────────────────────────────────────────────────────────────┐ │ IN-SAMPLE (Train & Test) │ │ Jan ─────────────────────────────────────────────────── Dec │ └────────────────────────────────────────────────────────────────────────┘ Result: 65% win rate (but is it real or overfitted?) Walk-Forward (Rolling Out-of-Sample = Realistic Performance): Window 1: ┌─────────────────────────────┬─────────────┐ │ TRAIN (Jan-Jun) │ TEST (Jul) │ → 58% win rate └─────────────────────────────┴─────────────┘ Window 2: ┌─────────────────────────────┬─────────────┐ │ TRAIN (Feb-Jul) │ TEST (Aug) │ → 61% win rate └─────────────────────────────┴─────────────┘ Window 3: ┌─────────────────────────────┬─────────────┐ │ TRAIN (Mar-Aug) │ TEST (Sep) │ → 55% win rate └─────────────────────────────┴─────────────┘ ...continues rolling forward... Walk-Forward Efficiency = Out-of-Sample Performance / In-Sample Performance Target: > 50% efficiency indicates strategy is robust 
var wfa = new WalkForwardAnalyzer(strategy);

var results = wfa.Analyze(
    startDate: DateTime.Parse("2025-01-01"),
    endDate: DateTime.Parse("2025-12-31"),
    trainingDays: 60,
    testingDays: 20
);

Console.WriteLine($"WF Efficiency: {results.Efficiency:P1}");

Common Pitfalls

🚫 Backtesting Biases to Avoid
Look-Ahead Bias Using future data that wouldn't be available at trade time
Survivorship Bias Only testing on stocks that still exist (ignoring delisted ones)
Curve Fitting Over-optimizing parameters to fit historical data perfectly
Ignoring Costs Not accounting for commissions, slippage, and spreads
Cherry-Picking Only showing results from favorable time periods

Best Practices

💡 Pro Tips
  • Test multiple market conditions - Bull markets, bear markets, sideways
  • Use realistic costs - Include commission + estimated slippage
  • Reserve data for validation - Don't optimize on your entire dataset
  • Be skeptical of perfect results - If it looks too good, it's probably overfit
  • Paper trade before live - Validate backtest results in real-time
