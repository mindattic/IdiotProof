Future-Proofing Architecture

Modern UI/backend communication strategies for seamless integration between IdiotProof.Core and friendly user interfaces.

The IPC Problem

Previous Attempts: Earlier versions used MAUI and Blazor with an "IPC server" paradigm that proved clunky. Running the backend as a separate process required complex inter-process communication, message serialization, and connection management.

The key insight: Don't run the backend as a separate process. Host everything in one app and use dependency injection. That's what made previous attempts clunky.

Recommended: In-Process Integration

Implementation Example

Blazor Web App (.NET 8+ Unified Model)

// Program.cs - One app, one process
var builder = WebApplication.CreateBuilder(args);

// Register your services ONCE
builder.Services.AddSingleton<IbWrapper>();
builder.Services.AddSingleton<TradeTrackingService>();
builder.Services.AddSingleton<StrategyRunner>();

// Add Blazor
builder.Services.AddRazorComponents()
    .AddInteractiveServerComponents();

var app = builder.Build();
app.MapRazorComponents<App>()
   .AddInteractiveServerRenderMode();

app.Run();

Our Recommendations

For Desktop Trading App

Blazor Hybrid with MAUI or WPF

  • Zero IPC - services run in-process
  • All C# (consistent with IdiotProof.Core)
  • Native performance for trading operations
  • BlazorWebView gives you modern web UI

For Web-Based Dashboard

Blazor Server (Interactive Server mode)

  • Zero IPC - all server-side
  • Real-time updates via SignalR (perfect for price feeds)
  • All C# on server
  • Access from any browser

Key Principle

Don't run the backend as a separate process.

Host everything in one application and use dependency injection. Services are registered once and injected wherever needed - no serialization, no protocols, no connection management.

Implementation Roadmap

  1. Phase 1: Create IdiotProof.BlazorUI class library with Razor components
  2. Phase 2: Build Blazor Server host that references IdiotProof.Core
  3. Phase 3: Wire up services via DI (no new code needed in Core)
  4. Phase 4: Add real-time updates using Blazor's built-in mechanisms
  5. Phase 5: Optionally wrap in MAUI shell for desktop deployment
