Pattern Matcher

LSH-based system for finding historical price patterns similar to current market conditions in sub-millisecond time.

What is Pattern Matching?

The Pattern Matcher uses Locality-Sensitive Hashing (LSH) to find historical price patterns that are similar to the current market state. By analyzing what happened after similar patterns in the past, it helps validate trading decisions.

PATTERN MATCHING CONCEPT ═══════════════════════════════════════════════════════════════════════════════ CURRENT PATTERN (Last 180 minutes) ───────────────────────────────────────────────────────────────────────── _____/\ ___/ \_ ____/ \____ \_____ "I've seen this before..." HISTORICAL MATCHES FOUND ───────────────────────────────────────────────────────────────────────── Match 1 (87% similar): Match 2 (82% similar): _____/\ ____/\ ___/ \_ ____/ \_ / \____ / \___ \_____/\ \___/\ ↑ ↑ +4.2% +3.1% (Outcome) (Outcome) Match 3 (79% similar): Match 4 (75% similar): ____/\ ___/\ ____/ \___ ____/ \ / \ / \____ \__/ \___ ↑ ↑ -1.8% +2.3% (Outcome) (Outcome) CONCLUSION: 3 of 4 similar patterns led to gains Average outcome: +1.95% Confidence: 75%

How LSH Works

Traditional similarity search is O(n) - you must compare against every historical pattern. LSH uses hash functions to eliminate most comparisons, achieving near O(1) lookup.

LOCALITY-SENSITIVE HASHING ═══════════════════════════════════════════════════════════════════════════════ TRADITIONAL SEARCH (Slow) LSH SEARCH (Fast) Query: [0.5, 0.3, 0.8, ...] Query: [0.5, 0.3, 0.8, ...] │ │ ▼ ▼ ┌──────────────────┐ ┌──────────────────┐ │ Compare vs ALL │ │ Hash query │ │ 10,000 patterns │ │ → Bucket 47 │ │ │ └────────┬─────────┘ │ Pattern 1: 45% │ │ │ Pattern 2: 23% │ ▼ │ Pattern 3: 87% ✓ │ ┌──────────────────┐ │ Pattern 4: 12% │ │ Bucket 47 has │ │ Pattern 5: 78% ✓ │ │ only 15 patterns │ │ ... │ │ (similar hash) │ │ Pattern 10000 │ └────────┬─────────┘ └──────────────────┘ │ │ ▼ 10,000 ┌──────────────────┐ comparisons │ Compare vs 15 │ │ Pattern 3: 87% ✓ │ │ Pattern 5: 78% ✓ │ └──────────────────┘ │ 15 comparisons THE KEY INSIGHT ═══════════════════════════════════════════════════════════════════════════════ LSH hash functions are designed so that: • SIMILAR vectors → LIKELY to hash to the SAME bucket • DISSIMILAR vectors → LIKELY to hash to DIFFERENT buckets This is the opposite of cryptographic hashes (which maximize collision avoidance).

Hyperplane-Based Hashing

IdiotProof uses random hyperplane LSH for cosine similarity. Each hyperplane divides the feature space in half, and the hash is which side of each plane the vector falls on.

RANDOM HYPERPLANE HASHING ═══════════════════════════════════════════════════════════════════════════════ 2D Example (actual system uses 10+ dimensions): Hyperplane 1 │ ┌────────────│────────────┐ │ A │ B │ │ (1,0) │ (1,1) │ ────────┼────────────+────────────┼──── Hyperplane 2 │ C │ D │ │ (0,0) │ (0,1) │ └────────────│────────────┘ │ Hash = [side of H1] [side of H2] Point in quadrant A → Hash = "10" Point in quadrant D → Hash = "01" Similar points (nearby) → same quadrant → same hash → found quickly! MULTI-DIMENSIONAL FEATURE VECTOR ═══════════════════════════════════════════════════════════════════════════════ Pattern features: Dimension 1: VWAP distance (normalized) Dimension 2: EMA(9) distance Dimension 3: RSI value Dimension 4: MACD histogram Dimension 5: ADX Dimension 6: +DI - -DI Dimension 7: Volume ratio Dimension 8: Price momentum Dimension 9: Volatility (ATR) Dimension 10: Price change % → 10D vector hashed using N hyperplanes → Hash determines bucket → Only compare within bucket

Pattern Storage

Patterns are stored per-ticker and persist across sessions.

PATTERN DATA STRUCTURE ═══════════════════════════════════════════════════════════════════════════════ Each stored pattern contains: ┌─────────────────────────────────────────────────────────────────────────┐ │ Pattern │ │ ├── Timestamp: When pattern occurred │ │ ├── Features: 10D feature vector │ │ ├── Hash: LSH bucket identifier │ │ ├── Candles: Last 180 minutes of OHLC data │ │ ├── Indicators: All indicator values at pattern time │ │ └── Outcome: │ │ ├── PriceChange5Min: +0.3% │ │ ├── PriceChange15Min: +1.2% │ │ ├── PriceChange30Min: +2.1% │ │ ├── MaxDrawdown: -0.5% │ │ └── MaxGain: +2.8% │ └─────────────────────────────────────────────────────────────────────────┘ Storage location: Data/{TICKER}.patterns.json

Pattern Match Result

PATTERN MATCH OUTPUT ═══════════════════════════════════════════════════════════════════════════════ PatternMatchResult { MatchCount // Number of similar patterns found AverageSimilarity // 0-1 average match quality BullishCount // Patterns that led to gains BearishCount // Patterns that led to losses AverageOutcome // Average price change % Confidence // 0-1 confidence in prediction TopMatches[] { Similarity // 0-1 how similar Outcome // What happened after Timestamp // When it occurred } } HOW CONFIDENCE IS CALCULATED ═══════════════════════════════════════════════════════════════════════════════ Confidence = f(MatchCount, Similarity, Agreement) Example: ───────────────────────────────────────────────────────────────────────── Found 8 matches with > 70% similarity 6 bullish outcomes, 2 bearish outcomes Agreement = 6/8 = 75% Average similarity = 78% Confidence = 0.75 × 0.78 = 0.585 (58.5%) If confidence > 50%, pattern supports entry If confidence < 50%, pattern suggests caution

Integration with Trading

Component How Pattern Matcher Helps
Entry Validation Before entering, check if similar patterns were profitable.
High confidence (>60%) = proceed, Low (<40%) = caution
TP/SL Sizing AverageOutcome and MaxGain help calibrate take profit targets.
MaxDrawdown helps set appropriate stop losses.
Exit Timing PriceChange at different intervals helps estimate when to exit.
If patterns fade after 15 minutes, don't hold for an hour.
